# UserService

A Spring Boot service for storing messages with optional sentiment analysis powered by Google Gemini.

## Tech Stack

Spring Boot · Kotlin · H2 (file-based) · Google Gemini API · ktlint

## Getting Started

1. Set your Gemini API key in `application.properties`:
   ```properties
   gemini.api.key=YOUR_KEY_HERE
   ```
   (API keys: https://aistudio.google.com/app/api-keys)

2. Run the service:
   ```bash
   ./mvnw spring-boot:run
   ```

## API Reference

### Create a message

```bash
curl -X POST http://localhost:8080/api/v1/messages \
  -H "Content-Type: application/json" \
  -d '{"text": "Hello world", "sentiment": "NEUTRAL"}'
```

The `sentiment` field is optional — if omitted, the Gemini API will analyze it automatically.
Returns `201 Created` with the saved message in the body and a `Location` header pointing to `/api/v1/messages/{id}`.

### List all messages

```bash
curl http://localhost:8080/api/v1/messages
```

### Get a message by ID

```bash
curl http://localhost:8080/api/v1/messages/<id>
```

Returns `404` if the message does not exist.

### Message object

| Field       | Type            | Required | Description                      |
|-------------|-----------------|----------|----------------------------------|
| text        | string          | yes      | The message content              |
| sentiment   | Sentiment or null | no     | One of: `HAPPY` or `UNKNOWN`      |
| id          | string (UUID)    | no     | Auto-generated by the service    |

## Loading Data

A CSV bulk-loader is available at `scripts/load-messages.py`.

**Setup:**
```bash
pip install -r scripts/requirements.txt
```

**Usage:**
```bash
# First column: message text
# Second column (optional): sentiment value from Sentiment.kt
python scripts/load-messages.py scripts/sample_messages.csv
```
